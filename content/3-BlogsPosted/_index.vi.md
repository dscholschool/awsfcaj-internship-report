###  [Blog 1 - Monitoring and Managing Claude Code Costs on Amazon Bedrock: A Comprehensive Guide for Developers](3.1-Blog1/)
Blog này giới thiệu kiến trúc giám sát toàn diện cho Claude Code khi triển khai trong đội ngũ phát triển. Bài viết hướng dẫn cách thiết lập hệ thống thu thập dữ liệu qua OpenTelemetry để theo dõi chi phí sử dụng token, đo lường hiệu suất lập trình, và đặc biệt là cơ chế quản lý ngân sách (quota) tự động nhằm ngăn chặn lãng phí tài nguyên AWS.

###  [Blog 2 - AWS Weekly Quick Summary: Claude Opus 4.8, Aurora MySQL + Kiro, AWS Transform, and Many Notable Updates](3.2-Blog2/)
Blog này tổng hợp các cập nhật quan trọng từ hệ sinh thái AWS, nổi bật với việc tích hợp AI vào mọi giai đoạn phát triển phần mềm. Các điểm nhấn bao gồm sự xuất hiện của mô hình Claude Opus 4.8 trên Amazon Bedrock, khả năng quản lý database tự nhiên qua Kiro trên Aurora MySQL, công cụ đánh giá migration thông minh trong AWS Transform, và OpenSearch Serverless được tối ưu hóa cho các Agentic AI.

###  [Blog 3 - Learn how Amazon Cognito automatically synchronizes authentication data across multiple regions to ensure application continuity during infrastructure outrages.](3.3-Blog3/)
Blog này phân tích tính năng Multi-Region Replication mới của Amazon Cognito, cho phép tự động đồng bộ hóa hồ sơ người dùng, thông tin xác thực và cấu hình User Pool giữa các khu vực AWS. Đây là giải pháp then chốt giúp doanh nghiệp xây dựng kiến trúc phục hồi sau thảm họa (Disaster Recovery) hiệu quả, đảm bảo trải nghiệm người dùng liền mạch và đáp ứng các tiêu chuẩn bảo mật nghiêm ngặt thông qua khả năng tùy chỉnh mã hóa với Customer Managed Key (CMK).
